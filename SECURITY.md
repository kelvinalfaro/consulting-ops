# Security and privacy

Do not open a public issue containing credentials, private company records, confidential client information, or proprietary solicitation documents.

Before publishing or sharing a repository copy, run the tests and inspect the Git staging inventory. User-layer files are ignored by default, but users remain responsible for confirming that generated reports, proposal workspaces, source documents, and local configuration are not staged.

Report a suspected code vulnerability privately to the repository maintainer through GitHub's private vulnerability reporting when available.
