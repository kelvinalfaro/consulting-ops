# Live submission preparation assistant

This is the consulting-native equivalent of career-ops apply. It prepares and, when explicitly supervised, may help transcribe a response. It never performs the final action.

## Preconditions

- Require the authoritative solicitation and amendments, a reviewed proposal workspace, completed compliance matrix, current attachment set, and confirmed submission channel.
- Run proposal quality and submission readiness first. Unchecked requirements, unknown certifications, unresolved representations, missing attachments, or conflicting portal instructions are hard stops.
- Generate `node consulting-ops.mjs apply <workspace>` and use the field pack as the working record.

## Field-by-field protocol

1. Read the live portal or delivery instructions without bypassing access controls.
2. Transcribe every field label, required marker, character/file limit, accepted format, certification, representation, signature, and final-action control.
3. Map each entry to the reviewed proposal, compliance matrix, approved firm source, or authorized current-session fact. Never infer an answer.
4. Show draft answers and file selections to the user in logical groups. Flag truncation, source conflicts, changed requirements, and portal-only terms.
5. Require the appropriate human owner for pricing, legal terms, insurance, certifications, representations, signatures, and attestations.
6. Save progress and a review record without storing passwords, tokens, or unnecessary sensitive data.

## Final stop

The assistant may navigate and populate fields only when the user explicitly requests supervised browser help. It must stop before every final Submit, Send, Certify, Sign, Accept, or equivalent control, show the final review checklist, and hand control to the user.
