# Interactive finalist practice

Load the solicitation, amendments, submitted proposal, evaluation criteria and weights, approved evidence, team roles, and confirmed event format. Hard-stop if the actual submitted response or event instructions are unavailable.

Generate a worksheet with `node consulting-ops.mjs finalist-practice <opportunity.yml>`. Build questions across understanding of need, approach, work plan, facilitation/coaching stance, team roles, past performance, risk, equity and participation, schedule, budget assumptions, difficult scenarios, and criterion-specific concerns.

Run the session one question at a time:

1. Ask the question and wait for the complete answer.
2. Record timing and the evidence invoked.
3. Give concise feedback on responsiveness, structure, specificity, credibility, unsupported claims, commitments, buyer language, and delivery.
4. Offer one improved answer structure using only approved evidence; do not script invented facts.
5. Ask the user to retry when improvement would materially help, then continue.

Track recurring gaps, weak handoffs, overlong answers, new commitments, and questions requiring procurement clarification. End with prioritized rehearsal actions and owners. Do not contact the issuer or alter the submitted proposal.
