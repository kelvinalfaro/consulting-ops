# Governance

The maintainer sets release scope, accepts contributions, and protects the project's safety invariants. Decisions favor portable local workflows, auditable source handling, backward-compatible data contracts, and human control over communications and submission.

Substantial changes should include tests, documentation, migration impact, and a clear explanation of how user-layer data remains protected. Security reports follow [SECURITY.md](SECURITY.md). Routine changes use pull requests; maintainers may revert changes that violate the trust boundary.
